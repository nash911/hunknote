> **Target**: Integrate a multi-phase LLM agent pipeline into hunknote's `compose` command, activated via `--agent` flag.
> **Codebase**: Python CLI tool using Typer, Pydantic, multi-provider LLM support (7 providers).
> **Existing compose**: Single-shot LLM call in `hunknote/cli/compose.py`. The agent replaces only the plan generation logic; parsing, inventory, caching, execution, and rendering are reused.

## 1. Architecture Overview

```
User runs: hunknote compose --agent

CLI (cli/compose.py)
  ├── Shared setup: parse_unified_diff, build_hunk_inventory, cache check
  ├── if --agent:
  │     └── AgentOrchestrator.run()
  │           ├── Phase 1:  Hunk Summarization (batched LLM calls, per-file minimum)
  │           ├── Phase 2:  Dependency Graph (ReAct agent with tools)
  │           ├── Phase 3:  Clustering (single LLM call)
  │           ├── Phase 4:  Ordering (topological sort + LLM tie-break)
  │           ├── Phase 5a: Validation (git worktree, layered checks)
  │           │     └── On failure → Phase 5b: Diagnosis (ReAct agent)
  │           │           └── Emit remediation → Orchestrator applies → retry from Phase 3/4
  │           └── Phase 6:  Commit Message Generation (single LLM call per commit)
  └── else:
        └── Existing single-shot LLM flow (unchanged)

All phases emit trace events → AgentTrace collects them
  → --trace flag displays full trace to stderr
  → Trace JSON always saved to .hunknote/agent_trace.json (cleared on commit)
  → --debug retains its current compose debug behavior (unchanged)
```

**Key principles:**
- The orchestrator is deterministic Python code (state machine). It never calls the LLM directly — it delegates to phase functions.
- ReAct agents (Phase 2, Phase 5b) are LLM loops that use tools and emit structured JSON.
- Every LLM output is validated by a deterministic gate before the orchestrator acts on it.
- Validated commits are frozen as read-only context; only mutable hunks are passed to reclustering.
- All LLM calls go through LiteLLM — a single unified interface for all 7 providers. No per-provider `generate_raw()` implementations needed.
- Every phase logs **complete** inputs and outputs to the trace (not truncated). The trace is a structured, nested JSON document saved to `.hunknote/agent_trace.json`.

## 2. Naming Conventions

These conventions are used throughout the codebase and must be maintained by the agent pipeline:

- **Hunk IDs**: `H{n}_{6-char-hash}` — e.g. `H1_abc123`, `H2_456def`, `H17_e4f347`. The numeric prefix is sequential starting from 1. The hash suffix is the first 6 characters of an MD5 of the hunk content. These are generated by the existing `compose/parser.py` and must be used as-is throughout the agent pipeline. The agent never creates new hunk IDs.
- **Commit IDs**: `C{n}` — e.g. `C1`, `C2`, `C3`. Sequential starting from 1. Assigned during Phase 4 (ordering) when commit groups receive their final order. During Phase 3 (clustering), groups use temporary IDs (`_G1`, `_G2`, ...) which are renamed to `C{n}` after ordering.

## Module 1: Data Models

**File**: `hunknote/compose/agent/models.py`

These models hold all intermediate state produced by each phase. The orchestrator owns instances of these and passes them between phases.

```python
from dataclasses import dataclass, field
from enum import Enum
from typing import Optional
from pydantic import BaseModel


# ── Phase 1 output ──

@dataclass
class HunkSummary:
    """Output of Phase 1 for a single hunk."""
    hunk_id: str               # e.g. "H1_abc123" — uses existing IDs from parser
    file_path: str
    intent: str                # e.g. "Add retry logic with exponential backoff"
    category: str              # "feature", "bugfix", "refactor", "test", "docs", "config"
    symbols_modified: list[str]    # Symbols added/removed/changed by this hunk
    symbols_referenced: list[str]  # Symbols used but not defined by this hunk
    is_new_file: bool = False      # True if this hunk belongs to a newly added file


# ── Phase 2 output ──

class EdgeType(Enum):
    DIRECTIONAL = "directional"      # A needs B first; B is valid without A
    BIDIRECTIONAL = "bidirectional"   # A and B must be in the same commit

@dataclass
class DependencyEdge:
    from_hunk: str       # e.g. "H3_abc123"
    to_hunk: str         # e.g. "H7_def456"
    edge_type: EdgeType
    reason: str

@dataclass
class DependencyGraph:
    edges: list[DependencyEdge] = field(default_factory=list)
    warnings: list[str] = field(default_factory=list)

    def add_edge(self, edge: DependencyEdge) -> None: ...
    def add_bidirectional(self, hunk_ids: list[str], reason: str) -> None: ...
    def get_edges_for_hunk(self, hunk_id: str) -> list[DependencyEdge]: ...
    def get_bidirectional_groups(self) -> list[set[str]]: ...
    def get_dependencies_of(self, hunk_id: str) -> set[str]: ...


# ── Phase 3 output ──

@dataclass
class CommitGroup:
    """A candidate commit. Uses temporary _G{n} IDs during Phase 3,
    renamed to C{n} during Phase 4 ordering."""
    group_id: str           # "_G1" during clustering, "C1" after ordering
    hunk_ids: list[str]     # e.g. ["H1_abc123", "H5_def456"]
    theme: str              # Short description of what this group does
    category: str           # "feature", "refactor", "test", etc.


# ── Phase 5a output ──

class ValidationLayer(Enum):
    PATCH_APPLY = "patch_apply"
    SYNTAX = "syntax"
    COMPILE = "compile"
    IMPORT = "import"
    TEST = "test"

@dataclass
class ValidationFailure:
    commit_index: int
    commit_group: CommitGroup
    layer: ValidationLayer
    error_output: str
    file_path: Optional[str] = None  # The file that failed, if applicable


# ── Phase 5b output ──

class RemediationAction(Enum):
    RECLUSTER = "recluster"
    REORDER = "reorder"
    SPLIT_COMMIT = "split_commit"
    UNLOCK_AND_RECLUSTER = "unlock_and_recluster"
    FAIL = "fail"

@dataclass
class Remediation:
    action: RemediationAction
    diagnosis: str
    modifications: dict = field(default_factory=dict)
    # modifications may contain:
    #   "merge_hunks_into_same_commit": list[list[str]]  e.g. [["H3_abc123", "H7_def456"]]
    #   "add_dependency_edges": list[dict]
    #   "split_commit_index": int
    #   "split_into": list[list[str]]
    revalidate_from_commit_index: int = 0
    unlock_from_commit_index: Optional[int] = None  # Only for unlock_and_recluster


# ── Frozen/Mutable state ──

@dataclass
class FrozenCommit:
    """A validated commit that should not be modified."""
    index: int
    message: str
    hunk_ids: list[str]              # e.g. ["H1_abc123", "H2_def456"]
    symbols_introduced: list[str]
    symbols_removed: list[str]

@dataclass
class FrozenBaseline:
    commits: list[FrozenCommit] = field(default_factory=list)

    @property
    def all_hunk_ids(self) -> set[str]:
        """Return all hunk IDs consumed by frozen commits."""
        ids = set()
        for c in self.commits:
            ids.update(c.hunk_ids)
        return ids

@dataclass
class MutableWorkingSet:
    hunk_ids: list[str] = field(default_factory=list)
    hunk_summaries: dict[str, HunkSummary] = field(default_factory=dict)
    # Dependency edges only among mutable hunks
    dependency_edges: list[DependencyEdge] = field(default_factory=list)
```

**Implementation notes:**
- `DependencyGraph` methods are simple list/set operations. `get_bidirectional_groups()` should use union-find to compute connected components of bidirectional edges.
- All models use `dataclass`. Pydantic is only used when parsing raw LLM JSON output (done inline in each phase, not in these models).

## Module 2: Tracing Infrastructure

**File**: `hunknote/compose/agent/tracing.py`

Every phase, every tool call, and every LLM call must emit trace events. The trace is:
1. A **nested** JSON tree (not flat log lines) — mirrors the pipeline call hierarchy.
2. Saved to `.hunknote/agent_trace.json` **incrementally** after each phase completes.
3. Cleared when the compose plan is committed (via `invalidate_compose_cache`).
4. Displayed on stderr when `--trace` flag is used (verbose mode).
5. The `--debug` flag retains its **current existing behavior** — showing compose debug info, NOT the agent trace.
6. **Complete data is stored** — full LLM prompts, full LLM responses, full tool outputs. Nothing is truncated in the trace file. The trace JSON file may be several MB for complex plans; this is acceptable for a debug artifact.

```python
import json
import time
from dataclasses import dataclass, field
from enum import Enum
from pathlib import Path
from typing import Any, Optional


class TraceEventType(Enum):
    PHASE_START = "phase_start"
    PHASE_END = "phase_end"
    LLM_CALL = "llm_call"
    LLM_RESPONSE = "llm_response"
    TOOL_CALL = "tool_call"
    TOOL_RESULT = "tool_result"
    VALIDATION_CHECK = "validation_check"
    VALIDATION_FAIL = "validation_fail"
    VALIDATION_PASS = "validation_pass"
    GATE_CHECK = "gate_check"
    GATE_VIOLATION = "gate_violation"
    GATE_PASS = "gate_pass"
    REMEDIATION = "remediation"
    STATE_CHANGE = "state_change"
    BATCH_PLAN = "batch_plan"
    WARNING = "warning"
    ERROR = "error"


@dataclass
class TraceEvent:
    event_type: TraceEventType
    phase: str                          # "phase1", "phase2", "phase5a", "phase5b", etc.
    message: str                        # Human-readable description
    timestamp: float = field(default_factory=time.time)
    duration_ms: Optional[float] = None
    tokens: Optional[dict[str, int]] = None  # {"input": N, "output": M, "thinking": K}
    # Complete data payloads — NEVER truncated in the trace.
    # LLM_CALL: {"system_prompt": <full>, "user_prompt": <full>, "model": "..."}
    # LLM_RESPONSE: {"raw_response": <full>, "model": "..."}
    # TOOL_CALL: {"tool_name": "...", "args": {...}}
    # TOOL_RESULT: {"tool_name": "...", "output": <full>, "success": bool}
    data: Optional[dict[str, Any]] = None
    children: list["TraceEvent"] = field(default_factory=list)


class AgentTrace:
    """Collects trace events from all phases of the agent pipeline.

    Produces a nested tree:
      orchestrator_run
        ├── phase1
        │     ├── batch_plan
        │     ├── llm_call (batch 1)
        │     ├── llm_response (batch 1)
        │     └── ...
        ├── phase2
        │     ├── llm_call (ReAct iteration 1)
        │     ├── llm_response
        │     ├── tool_call (ripgrep)
        │     ├── tool_result
        │     └── ...
        ├── phase5a
        │     ├── validation_check (C1, patch_apply)
        │     ├── validation_pass (C1)
        │     ├── validation_check (C2, compile)
        │     └── validation_fail (C2)
        └── ...
    """

    def __init__(self):
        self.root: TraceEvent = TraceEvent(
            event_type=TraceEventType.PHASE_START,
            phase="orchestrator",
            message="Agent pipeline started",
        )
        self._stack: list[TraceEvent] = [self.root]
        self._total_input_tokens: int = 0
        self._total_output_tokens: int = 0
        self._total_thinking_tokens: int = 0
        self._total_llm_calls: int = 0

    @property
    def _current_parent(self) -> TraceEvent:
        return self._stack[-1]

    def _append(self, event: TraceEvent) -> None:
        self._current_parent.children.append(event)

    def phase_start(self, phase: str, message: str = "") -> None:
        """Open a new phase scope. Subsequent events nest under it."""
        event = TraceEvent(
            event_type=TraceEventType.PHASE_START,
            phase=phase,
            message=message or f"Starting {phase}",
        )
        self._append(event)
        self._stack.append(event)

    def phase_end(self, phase: str, message: str = "") -> None:
        """Close the current phase scope and record duration."""
        if len(self._stack) > 1 and self._stack[-1].phase == phase:
            started = self._stack[-1]
            started.duration_ms = (time.time() - started.timestamp) * 1000
            started.event_type = TraceEventType.PHASE_END
            if message:
                started.message = message
            self._stack.pop()

    def llm_call(self, phase: str, system_prompt: str, user_prompt: str,
                 model: str = "") -> None:
        """Record a complete LLM call with FULL prompts (not truncated)."""
        self._total_llm_calls += 1
        self._append(TraceEvent(
            event_type=TraceEventType.LLM_CALL,
            phase=phase,
            message=f"LLM call #{self._total_llm_calls}",
            data={
                "system_prompt": system_prompt,
                "user_prompt": user_prompt,
                "model": model,
            },
        ))

    def llm_response(self, phase: str, raw_response: str, model: str,
                     tokens: dict[str, int], duration_ms: float) -> None:
        """Record a complete LLM response with FULL output (not truncated)."""
        self._total_input_tokens += tokens.get("input", 0)
        self._total_output_tokens += tokens.get("output", 0)
        self._total_thinking_tokens += tokens.get("thinking", 0)
        self._append(TraceEvent(
            event_type=TraceEventType.LLM_RESPONSE,
            phase=phase,
            message=f"LLM response from {model}",
            duration_ms=duration_ms,
            tokens=tokens,
            data={"raw_response": raw_response, "model": model},
        ))

    def tool_call(self, phase: str, tool_name: str, args: dict) -> None:
        self._append(TraceEvent(
            event_type=TraceEventType.TOOL_CALL,
            phase=phase,
            message=f"Tool call: {tool_name}",
            data={"tool_name": tool_name, "args": args},
        ))

    def tool_result(self, phase: str, tool_name: str, output: str,
                    success: bool, duration_ms: float) -> None:
        """Record tool result with FULL output (not truncated)."""
        self._append(TraceEvent(
            event_type=TraceEventType.TOOL_RESULT,
            phase=phase,
            message=f"Tool result: {tool_name} ({'ok' if success else 'fail'})",
            duration_ms=duration_ms,
            data={"tool_name": tool_name, "output": output, "success": success},
        ))

    def gate_violation(self, phase: str, violations: list[dict]) -> None:
        self._append(TraceEvent(
            event_type=TraceEventType.GATE_VIOLATION, phase=phase,
            message=f"Gate check failed: {len(violations)} violation(s)",
            data={"violations": violations},
        ))

    def gate_pass(self, phase: str, message: str = "") -> None:
        self._append(TraceEvent(
            event_type=TraceEventType.GATE_PASS, phase=phase,
            message=message or "Gate check passed",
        ))

    def state_change(self, phase: str, message: str, data: dict = None) -> None:
        self._append(TraceEvent(
            event_type=TraceEventType.STATE_CHANGE, phase=phase,
            message=message, data=data,
        ))

    def warning(self, phase: str, message: str, data: dict = None) -> None:
        self._append(TraceEvent(
            event_type=TraceEventType.WARNING, phase=phase,
            message=message, data=data,
        ))

    def batch_plan(self, phase: str, batches: list[dict]) -> None:
        self._append(TraceEvent(
            event_type=TraceEventType.BATCH_PLAN, phase=phase,
            message=f"Planned {len(batches)} batch(es)",
            data={"batches": batches},
        ))

    # ── Persistence ──

    def save_to_file(self, repo_root: Path) -> None:
        """Save the full trace tree to .hunknote/agent_trace.json.

        Called by the orchestrator after each phase completes (incremental save).
        If the pipeline crashes mid-way, the trace up to that point is preserved.
        """
        trace_file = repo_root / ".hunknote" / "agent_trace.json"
        trace_file.parent.mkdir(parents=True, exist_ok=True)

        def event_to_dict(event: TraceEvent) -> dict:
            d = {
                "event_type": event.event_type.value,
                "phase": event.phase,
                "message": event.message,
                "timestamp": event.timestamp,
            }
            if event.duration_ms is not None:
                d["duration_ms"] = round(event.duration_ms, 2)
            if event.tokens:
                d["tokens"] = event.tokens
            if event.data:
                d["data"] = event.data
            if event.children:
                d["children"] = [event_to_dict(c) for c in event.children]
            return d

        trace_data = {
            "version": "1",
            "summary": self.get_summary(),
            "trace": event_to_dict(self.root),
        }
        trace_file.write_text(json.dumps(trace_data, indent=2, default=str))

    @staticmethod
    def clear_trace_file(repo_root: Path) -> None:
        """Delete the trace file. Called when the compose plan is committed."""
        trace_file = repo_root / ".hunknote" / "agent_trace.json"
        if trace_file.exists():
            trace_file.unlink()

    # ── Reporting ──

    def get_summary(self) -> dict:
        return {
            "total_llm_calls": self._total_llm_calls,
            "total_input_tokens": self._total_input_tokens,
            "total_output_tokens": self._total_output_tokens,
            "total_thinking_tokens": self._total_thinking_tokens,
            "total_duration_ms": round((time.time() - self.root.timestamp) * 1000, 2),
        }

    def format_for_stderr(self, verbose: bool = False) -> str:
        """Format the trace for stderr output.

        verbose=False (default): Compact progress lines:
            "Phase 1: 12 hunks summarized in 3 batches (1.2s) ✓"
            "Phase 2: 8 dependencies found (3.4s) ✓"
            "Phase 5a: Commit C3 failed compile check, retrying..."

        verbose=True (--trace flag): Full nested trace showing every
            LLM call (with full prompts/responses), tool call/result,
            gate check, and state change.
        """

```

**Implementation notes:**
- `AgentTrace` is instantiated by the orchestrator and passed to every phase function.
- In non-trace mode, only `format_for_stderr(verbose=False)` output is shown (compact progress lines).
- In `--trace` mode, `format_for_stderr(verbose=True)` shows the full nested trace with every LLM call (full prompts/responses), tool calls/results, gate checks, and state changes.
- Every LLM call logs: the FULL system prompt, the FULL user prompt, the FULL response, token counts, and wall-clock duration. Nothing is truncated in the trace file.
- Every tool call logs: tool name, args, FULL result output, and duration. The trace stores the complete untruncated output even though the LLM only sees the truncated version.
- The trace file is saved incrementally after each phase completes, so if the pipeline crashes mid-way, the trace up to that point is preserved.

## Module 3: Agent Tools

**File**: `hunknote/compose/agent/tools.py`

These are the tools available to the ReAct agents in Phase 2 and Phase 5b. Each tool is a Python function with a structured signature and a description string for the LLM prompt.

```python
import subprocess
import shutil
from dataclasses import dataclass
from pathlib import Path
from typing import Optional

from hunknote.compose.models import HunkRef


@dataclass
class ToolResult:
    """Structured result from a tool call."""
    tool_name: str
    success: bool
    output: str           # Text result shown to the LLM (may be truncated)
    full_output: str      # Complete untruncated output (for trace storage)
    truncated: bool = False


# Max chars returned to the LLM context from a single tool call.
# Full output is always stored in the trace via full_output field.
TOOL_OUTPUT_MAX_CHARS = 3000


# ── Tool Registry ──

TOOL_DEFINITIONS = [
    {
        "name": "ripgrep",
        "description": "Search the repository for a text pattern. Returns matching file paths, line numbers, and context.",
        "parameters": {
            "pattern": "string (required) — regex pattern to search for",
            "file_glob": "string (optional) — file glob filter, e.g. '*.py' or '*.ts'",
            "context_lines": "int (optional, default 2) — lines of context around matches",
            "max_results": "int (optional, default 20) — maximum matches to return",
        },
    },
    {
        "name": "read_file",
        "description": "Read lines from a file in the repository (current HEAD version).",
        "parameters": {
            "file_path": "string (required) — path relative to repo root",
            "line_start": "int (optional) — first line to read (1-indexed)",
            "line_end": "int (optional) — last line to read (inclusive)",
        },
    },
    {
        "name": "read_staged_file",
        "description": "Read the staged version of a file (HEAD + all staged hunks applied). Use this to see what the file WILL look like after all changes.",
        "parameters": {
            "file_path": "string (required) — path relative to repo root",
            "line_start": "int (optional)",
            "line_end": "int (optional)",
        },
    },
    {
        "name": "get_hunk",
        "description": "Retrieve the full diff content and metadata of a specific hunk by ID.",
        "parameters": {
            "hunk_id": "string (required) — the hunk ID, e.g. 'H3_a1b2c3'",
        },
    },
    {
        "name": "list_hunks",
        "description": "List all hunk IDs with their file paths and header lines. Use this for a quick overview.",
        "parameters": {},
    },
]


# ── Tool Implementations ──

def execute_tool(
    tool_name: str,
    args: dict,
    repo_root: Path,
    inventory: dict[str, HunkRef],
    hunk_summaries: Optional[dict] = None,
) -> ToolResult:
    """Execute a tool and return a structured result."""
    if tool_name == "ripgrep":
        return _run_ripgrep(repo_root, **args)
    elif tool_name == "read_file":
        return _run_read_file(repo_root, **args)
    elif tool_name == "read_staged_file":
        return _run_read_staged_file(repo_root, **args)
    elif tool_name == "get_hunk":
        return _run_get_hunk(inventory, hunk_summaries, **args)
    elif tool_name == "list_hunks":
        return _run_list_hunks(inventory)
    else:
        return ToolResult(tool_name=tool_name, success=False,
                          output=f"Unknown tool: {tool_name}",
                          full_output=f"Unknown tool: {tool_name}")


def _truncate(text: str, max_chars: int = TOOL_OUTPUT_MAX_CHARS) -> tuple[str, bool]:
    """Truncate for LLM context. Returns (truncated_text, was_truncated)."""
    if len(text) <= max_chars:
        return text, False
    return text[:max_chars] + "\n... [truncated]", True


def _run_ripgrep(repo_root: Path, pattern: str, file_glob: str = "",
                 context_lines: int = 2, max_results: int = 20) -> ToolResult:
    """Run ripgrep on the repo. Falls back to grep if rg is not installed."""
    cmd = ["rg", "--line-number", f"--context={context_lines}", f"--max-count={max_results}"]
    if file_glob:
        cmd.extend(["--glob", file_glob])
    cmd.append(pattern)

    result = subprocess.run(cmd, capture_output=True, text=True, cwd=repo_root, timeout=10)
    full_output = result.stdout
    if not full_output.strip():
        full_output = "(no matches found)"
    output, truncated = _truncate(full_output)
    return ToolResult(tool_name="ripgrep", success=True, output=output,
                      full_output=full_output, truncated=truncated)


def _run_read_file(repo_root: Path, file_path: str,
                   line_start: int = None, line_end: int = None) -> ToolResult:
    """Read lines from a file (HEAD version via git show HEAD:<path>)."""
    cmd = ["git", "show", f"HEAD:{file_path}"]
    result = subprocess.run(cmd, capture_output=True, text=True, cwd=repo_root, timeout=10)
    ...


def _run_read_staged_file(repo_root: Path, file_path: str,
                          line_start: int = None, line_end: int = None) -> ToolResult:
    """Read the staged version of a file (index version via git show :<path>)."""
    cmd = ["git", "show", f":{file_path}"]
    result = subprocess.run(cmd, capture_output=True, text=True, cwd=repo_root, timeout=10)
    ...


def _run_get_hunk(inventory: dict[str, HunkRef], hunk_summaries: Optional[dict],
                  hunk_id: str) -> ToolResult:
    """Return the full diff and summary of a hunk."""
    hunk = inventory.get(hunk_id)
    if not hunk:
        return ToolResult(tool_name="get_hunk", success=False,
                          output=f"Hunk {hunk_id} not found",
                          full_output=f"Hunk {hunk_id} not found")
    lines = "\n".join(hunk.lines)
    summary_text = ""
    if hunk_summaries and hunk_id in hunk_summaries:
        s = hunk_summaries[hunk_id]
        summary_text = f"\nSummary: {s.intent}\nCategory: {s.category}"
    full_output = f"Hunk {hunk_id} in {hunk.file_path}\n{hunk.header}\n{lines}{summary_text}"
    output, truncated = _truncate(full_output)
    return ToolResult(tool_name="get_hunk", success=True, output=output,
                      full_output=full_output, truncated=truncated)


def _run_list_hunks(inventory: dict[str, HunkRef]) -> ToolResult:
    """List all hunk IDs with file paths and headers."""
    lines = []
    for hid in sorted(inventory.keys(), key=lambda x: int(x.split("_")[0][1:])):
        h = inventory[hid]
        lines.append(f"{hid}: {h.file_path}  {h.header.strip()}")
    full_output = "\n".join(lines)
    output, truncated = _truncate(full_output)
    return ToolResult(tool_name="list_hunks", success=True, output=output,
                      full_output=full_output, truncated=truncated)
```

**Implementation notes:**
- `read_file` should use `git show HEAD:<path>` so it reads the committed version, not the working tree version. This is important because the working tree may have unstaged changes.
- `read_staged_file` uses `git show :<path>` to read from the index (staged version).
- `ToolResult` has both `output` (truncated for LLM context, max 3000 chars) and `full_output` (complete untruncated, for trace storage). The ReAct engine passes `output` to the LLM and `full_output` to `trace.tool_result()`.
- `ripgrep`: If `rg` is not on PATH, fall back to `grep -rn`. Check with `shutil.which("rg")`.
- Tool timeouts: All subprocess calls should have `timeout=10` to prevent hangs.

    1. Send the system prompt + accumulated conversation to the LLM.
    2. Parse the LLM response for either:
       a. A tool call (Action: tool_name, Args: {...}) → execute tool, append result, loop.
       b. A final output (Output: {...}) → parse JSON, return.
    3. If max iterations or tool calls exceeded, ask LLM to emit final output immediately.

    Args:
        config: ReAct configuration (max iterations, tools, prompts).
        initial_context: The initial user message with all context for the agent.

        repo_root: Path to the git repo root.

        trace: AgentTrace instance for logging.
        phase_name: Name of the current phase (for tracing).
        hunk_summaries: Optional dict of hunk_id -> HunkSummary (for get_hunk tool).

    Returns:
        ReActResult with the parsed final output.
    """
    ...
```

**Implementation notes:**

The ReAct prompt format should use a clear, parseable structure. The system prompt should include:

```
You have access to the following tools:
{tool_definitions_formatted}

To use a tool, respond with EXACTLY this format:
Thought: <your reasoning>
Action: <tool_name>
Args: {"param1": "value1", "param2": "value2"}

After receiving the tool result, continue reasoning.

When you have enough information to produce your final answer, respond with:
Thought: <final reasoning>
Output:
```json
<your structured JSON output>
```​
```

The parsing logic should:
1. Look for `Action:` and `Args:` lines → extract tool name and JSON args → call `execute_tool` → append result as `Observation:` → continue loop.
2. Look for `Output:` followed by a JSON block → parse JSON → return as final output.
3. If neither is found, treat the response as a thought and ask the LLM to continue with a tool call or final output.

The conversation is accumulated across iterations. Each iteration appends:
- The LLM's response (Thought + Action)
- The tool result (Observation: ...)

When approaching the max iteration limit (e.g., iteration >= max - 2), append a nudge message: "You are running low on iterations. Please produce your final Output now."

Token counting: Accumulate `input_tokens`, `output_tokens`, `thinking_tokens` across all LLM calls in the loop.

**Tracing**: Every iteration of the ReAct loop must:
- Call `trace.llm_call()` with the FULL system prompt and FULL accumulated user prompt BEFORE the LLM call.
- Call `trace.llm_response()` with the FULL raw response AFTER the LLM call.
- Call `trace.tool_call()` and `trace.tool_result()` (with `full_output`) for each tool invocation.

## Module 8: Phase 4 — Ordering

- The LLM output is just an ordered list of group IDs.
- **After ordering, rename group IDs from temporary `_G1`, `_G2` to final `C1`, `C2`, ...** based on their position in the sorted sequence.

## Module 9: Phase 5a — Validation (Git Worktree)

**File**: `hunknote/compose/agent/phases/validate.py`

Applies each commit's patch in an isolated git worktree and runs layered validation.

```python
def run_phase5a_validate(
    ordered_groups: list[CommitGroup],
    inventory: dict[str, HunkRef],
    file_diffs: list[FileDiff],
    repo_root: Path,
    trace: AgentTrace,
    start_from_index: int = 0,
    run_tests: bool = False,
) -> tuple[Optional[ValidationFailure], FrozenBaseline]:
    """Validate the commit sequence in an isolated git worktree.

    For each commit (starting from start_from_index):
      Layer 1: git apply — does the patch apply cleanly?
      Layer 2: tree-sitter parse — are touched files syntactically valid?
      Layer 3: Compile check — py_compile / python -c "import X" / tsc --noEmit / etc.
      Layer 4: (Optional) Test execution — run relevant tests.

    Bail on first failure.

    Worktree location: ~/.hunknote/tmp/hunknote_validate_{pid}
    (NOT /tmp/ — uses the user's hunknote directory for persistence and cleanup)

    Returns:
        (None, frozen_baseline) if all commits pass.
        (ValidationFailure, frozen_baseline_up_to_failure) if a commit fails.
    """
```

**Git worktree setup:**
```python
# Use ~/.hunknote/tmp/ for worktree isolation
worktree_base = Path.home() / ".hunknote" / "tmp"
worktree_base.mkdir(parents=True, exist_ok=True)
worktree_path = worktree_base / f"hunknote_validate_{os.getpid()}"

# Create detached worktree from HEAD
subprocess.run(["git", "worktree", "add", "--detach", str(worktree_path), "HEAD"],
               capture_output=True, text=True, cwd=repo_root)
```

**Per-commit validation loop (inside the worktree):**
```python
for i, group in enumerate(ordered_groups):
    if i < start_from_index:
        # Already validated — just apply the patch to advance state
        # (needed so subsequent commits apply on top of correct state)
        git_apply(patch)
        git_commit("simulation")
        continue

    # Layer 1: Patch applies?
    patch_content = build_commit_patch(group_as_planned_commit, inventory, file_diffs)
    result = subprocess.run(["git", "apply", "--check", patch_content_file], ...)
    if result.returncode != 0:
        return ValidationFailure(i, group, ValidationLayer.PATCH_APPLY, result.stderr)

    # Actually apply it
    subprocess.run(["git", "apply", patch_content_file], ...)

    # Layer 2: Syntax check (tree-sitter)
    for file_path in touched_files:
        # Run tree-sitter parse; check for ERROR nodes
        ...

    # Layer 3: Compile check (language-specific)
    for file_path in touched_files:
        ext = Path(file_path).suffix
        if ext == ".py":
            # py_compile
            subprocess.run(["python", "-m", "py_compile", file_path], ...)
            # import check: extract module name from file path
            module = file_path_to_module(file_path)
            subprocess.run(["python", "-c", f"import {module}"], ...)
        elif ext in (".ts", ".tsx"):
            subprocess.run(["npx", "tsc", "--noEmit"], ...)
        elif ext in (".go",):
            subprocess.run(["go", "build", "./..."], ...)
        # ... etc.

    # Commit in simulation worktree to advance state
    subprocess.run(["git", "add", "-A"], ...)
    subprocess.run(["git", "commit", "-m", f"sim-{i}"], ...)

    # Build frozen commit entry
    frozen_commit = FrozenCommit(
        index=i,
        message=group.theme,
        hunk_ids=group.hunk_ids,
        symbols_introduced=...,  # From hunk summaries
        symbols_removed=...,
    )
    frozen_baseline.commits.append(frozen_commit)
```

**Cleanup (always, even on failure):**
```python
subprocess.run(["git", "worktree", "remove", str(worktree_path), "--force"],
               capture_output=True, text=True, cwd=repo_root)
```

**Implementation notes:**
- The worktree is under `~/.hunknote/tmp/`, NOT `/tmp/`. This keeps temporary files in the user's hunknote directory for easier debugging and cleanup.
- The worktree is fully isolated. The user's working tree, index, and branches are never touched.
- For `start_from_index > 0` (re-validation after remediation), we still need to apply all prior patches to get the worktree to the right state. We just skip the validation checks for those.
- `file_path_to_module`: Convert `src/foo/bar.py` → `src.foo.bar`. Handle `__init__.py` files.
- Tree-sitter: If tree-sitter is not installed, skip Layer 2 with a warning. Do NOT make it a hard requirement.
- Compile checks: Only run for languages where the tool is available. Check `shutil.which("npx")`, `shutil.which("go")`, etc. Skip with a warning if not found.
- The `python -c "import X"` check needs to run inside the worktree directory with the right `PYTHONPATH` or `sys.path`. Use `cwd=worktree_path` and potentially set `PYTHONPATH=worktree_path`.
- Each validation layer result (pass or fail) should be recorded in the trace with `trace.validation_check()`, `trace.validation_pass()`, or `trace.validation_fail()`.
- Build `FrozenCommit` entries incrementally as each commit passes validation.

## Module 12: Orchestrator

**File**: `hunknote/compose/agent/orchestrator.py`

Deterministic state machine. Calls `trace.save_to_file(repo_root)` after every phase completes.

```python
class AgentPipelineError(Exception):
    def __init__(self, message: str, diagnosis: str = ""):
        super().__init__(message)
        self.diagnosis = diagnosis

@dataclass
class OrchestratorConfig:
    max_retries: int = 3
    max_commits: int = 6
    run_tests: bool = False
    style_config: object = None
    effective_profile: object = None

class AgentOrchestrator:
    def __init__(self, file_diffs, inventory, repo_root, llm_call_fn, config, trace=None):
        self.file_diffs = file_diffs
        self.inventory = inventory
        self.repo_root = repo_root
        self.llm_call_fn = llm_call_fn
        self.config = config
        self.trace = trace or AgentTrace()

        # Pipeline state
        self.summaries: dict[str, HunkSummary] = {}
        self.dependency_graph = DependencyGraph()
        self.commit_groups: list[CommitGroup] = []
        self.ordered_groups: list[CommitGroup] = []
        self.frozen_baseline = FrozenBaseline()
        self.planned_commits: list[PlannedCommit] = []
        self.retry_count = 0
        self.revalidate_from = 0

    def run(self) -> ComposePlan:
        """Execute the full agent pipeline and return a ComposePlan.

        This is the main entry point called from the CLI.

        Returns:
            ComposePlan compatible with the existing compose rendering/execution.

        Raises:
            AgentPipelineError: If the pipeline fails after max retries.
        """
        # High-level flow:
        # Phase 1 → Phase 2 → Phase 3 → Phase 4
        # Phase 5a + retry loop (Phase 5b on failure → apply remediation → retry)
        # Phase 6
        # save_to_file after each phase
        # Return ComposePlan

        # Phase 1
        self.trace.phase_start("phase1", f"Summarizing {len(self.inventory)} hunks")
        self.summaries = run_phase1_summarize(
            self.inventory, self.file_diffs, self.repo_root,
            self.llm_call_fn, self.trace,
        )
        self.trace.phase_end("phase1")

        # Phase 2
        self.trace.phase_start("phase2", "Building dependency graph")
        self.dependency_graph = run_phase2_dependency_graph(
            self.inventory, self.summaries, self.repo_root,
            self.llm_call_fn, self.trace,
        )
        self.trace.phase_end("phase2")

        # Phase 3
        all_hunk_ids = sorted(self.inventory.keys())
        self.trace.phase_start("phase3", "Clustering hunks into commits")
        self.commit_groups = run_phase3_cluster(
            self.summaries, self.dependency_graph,
            mutable_hunk_ids=all_hunk_ids,
            frozen_baseline=self.frozen_baseline,
            max_commits=self.config.max_commits,
            llm_call_fn=self.llm_call_fn,
            trace=self.trace,
        )
        self.trace.phase_end("phase3")

        # Phase 4
        self.trace.phase_start("phase4", "Ordering commits")
        self.ordered_groups = run_phase4_order(
            self.commit_groups, self.dependency_graph,
            self.frozen_baseline, self.llm_call_fn, self.trace,
        )
        self.trace.phase_end("phase4")

        # Phase 5 + retry loop
        while self.retry_count <= self.config.max_retries:
            self.trace.phase_start("phase5", f"Validating commit sequence (attempt {self.retry_count + 1})")
            failure, self.frozen_baseline = run_phase5_validate(
                self.ordered_groups, self.inventory, self.file_diffs,
                self.repo_root, self.trace,
                start_from_index=self.revalidate_from,
                run_tests=self.config.run_tests,
            )
            self.trace.phase_end("phase5")

            if failure is None:
                break  # All commits validated!

            if self.retry_count >= self.config.max_retries:
                raise AgentPipelineError(
                    f"Validation failed after {self.config.max_retries} retries",
                    diagnosis=f"Commit {failure.commit_index} failed at {failure.layer.value}: {failure.error_output[:500]}",
                )

            # Phase 5b: Diagnose and remediate
            self.trace.phase_start("phase5b", f"Diagnosing failure at commit {failure.commit_index}")

            mutable_ids = self._get_mutable_hunk_ids()
            remediation = run_phase5b_diagnose(
                failure, self.ordered_groups, self.dependency_graph,
                self.summaries, self.frozen_baseline, mutable_ids,
                self.inventory, self.repo_root, self.llm_call_fn, self.trace,
            )
            self.trace.phase_end("phase5b")

            if remediation.action == RemediationAction.FAIL:
                raise AgentPipelineError(
                    "Agent determined the failure cannot be resolved by regrouping",
                    diagnosis=remediation.diagnosis,
                )

            self._apply_remediation(remediation)
            self.retry_count += 1

        # Phase 6: Generate messages
        self.trace.phase_start("phase6", "Generating commit messages")
        self.planned_commits = run_phase6_messages(
            self.ordered_groups, self.inventory, self.summaries,
            self.config.style_config, self.config.effective_profile,
            self.llm_call_fn, self.trace,
        )
        self.trace.phase_end("phase6")

        # Convert to ComposePlan
        return ComposePlan(
            version="1",
            warnings=[w for w in self.dependency_graph.warnings],
            commits=self.planned_commits,
        )

    def _get_mutable_hunk_ids(self) -> list[str]:
        frozen = self.frozen_baseline.all_hunk_ids
        return [hid for hid in self.inventory.keys() if hid not in frozen]

     def _apply_remediation(self, remediation: Remediation) -> None:
        """Apply a remediation action: update graph, recluster, reorder."""

        self.trace.emit(TraceEvent(
            event_type=TraceEventType.REMEDIATION,
            phase="orchestrator",
            message=f"Applying {remediation.action.value}: {remediation.diagnosis}",
            data={"action": remediation.action.value, "modifications": remediation.modifications},
        ))
 
        if remediation.action == RemediationAction.UNLOCK_AND_RECLUSTER:
            # Unfreeze commits from the specified index onward
            unlock_from = remediation.unlock_from_commit_index or 0
            for commit in self.frozen_baseline.commits[unlock_from:]:
                pass  # Their hunk_ids return to the mutable set automatically via _get_mutable_hunk_ids()
            self.frozen_baseline.commits = self.frozen_baseline.commits[:unlock_from]
            self.revalidate_from = unlock_from

        # Apply graph modifications
        mods = remediation.modifications
        for edge_data in mods.get("add_dependency_edges", []):
            self.dependency_graph.add_edge(DependencyEdge(
                from_hunk=edge_data["from"],
                to_hunk=edge_data["to"],
                edge_type=EdgeType(edge_data["type"]),
                reason=edge_data.get("reason", ""),
            ))
        for group in mods.get("merge_hunks_into_same_commit", []):
            self.dependency_graph.add_bidirectional(group, reason="forced by remediation")

        # Re-cluster and re-order (mutable hunks only)
        if remediation.action in (RemediationAction.RECLUSTER, RemediationAction.UNLOCK_AND_RECLUSTER):
            mutable_ids = self._get_mutable_hunk_ids()
            self.commit_groups = run_phase3_cluster(
                self.summaries, self.dependency_graph,
                mutable_hunk_ids=mutable_ids,
                frozen_baseline=self.frozen_baseline,
                max_commits=self.config.max_commits,
                llm_call_fn=self.llm_call_fn,
                trace=self.trace,
            )
            self.ordered_groups = run_phase4_order(
                self.commit_groups, self.dependency_graph,
                self.frozen_baseline, self.llm_call_fn, self.trace,
            )
        elif remediation.action == RemediationAction.REORDER:
            self.ordered_groups = run_phase4_order(
                self.commit_groups, self.dependency_graph,
                self.frozen_baseline, self.llm_call_fn, self.trace,
            )
        elif remediation.action == RemediationAction.SPLIT_COMMIT:
            # Split the specified commit group
            split_idx = mods.get("split_commit_index", 0)
            split_into = mods.get("split_into", [])
            if split_into and split_idx < len(self.commit_groups):
                original = self.commit_groups[split_idx]
                new_groups = []
                for j, hunk_list in enumerate(split_into):
                    new_groups.append(CommitGroup(
                        group_id=f"{original.group_id}_{j}",
                        hunk_ids=hunk_list,
                        theme=f"Split {j+1} of {original.theme}",
                        category=original.category,
                    ))
                self.commit_groups = (
                    self.commit_groups[:split_idx]
                    + new_groups
                    + self.commit_groups[split_idx + 1:]
                )
            self.ordered_groups = run_phase4_order(
                self.commit_groups, self.dependency_graph,
                self.frozen_baseline, self.llm_call_fn, self.trace,
            )
 
        self.revalidate_from = remediation.revalidate_from_commit_index

        # Prepend frozen baseline's ordered groups back for full ordered list
        # (Phase 5 needs the full sequence to apply patches in order)
        frozen_groups = []
        for fc in self.frozen_baseline.commits:
            frozen_groups.append(CommitGroup(
                group_id=f"frozen_{fc.index}",
                hunk_ids=fc.hunk_ids,
                theme=fc.message,
                category="",
            ))
        self.ordered_groups = frozen_groups + self.ordered_groups
```

See full orchestrator logic in [Module 12: Orchestrator (detailed)](#module-12-orchestrator) in the previous version. The key flow:
- Phase 5a + retry loop: validate → on failure → Phase 5b diagnose → apply remediation → re-cluster/re-order → loop back to Phase 5a.
- `_apply_remediation` handles: `RECLUSTER`, `REORDER`, `SPLIT_COMMIT`, `UNLOCK_AND_RECLUSTER`, `FAIL`.
- After `UNLOCK_AND_RECLUSTER`: truncate `frozen_baseline.commits` at `unlock_from_commit_index`, all hunks from unlocked commits return to mutable set automatically.
- Frozen groups are prepended to `ordered_groups` for Phase 5a (needs full sequence to apply patches).

## Module 13: LiteLLM Integration

**File**: `hunknote/compose/agent/llm.py`

Instead of implementing `generate_raw()` on each of the 7 provider classes, the agent uses **LiteLLM** as a unified interface. This eliminates all per-provider boilerplate for the agent pipeline.

**Why LiteLLM:**
- Single `litellm.completion()` call wraps all 7 providers with identical input/output.
- Handles API differences transparently (Anthropic's `system=`, Cohere's chat endpoint, etc.).
- Built-in retry logic, timeout handling, error normalization.
- Adding future providers is zero-code.
- Existing per-provider classes in `hunknote/llm/` are untouched — LiteLLM is only used by the agent.

**Installation**: Add `litellm` to project dependencies (`pyproject.toml`).

```python
import time
import litellm
from hunknote.llm.base import RawLLMResult
from hunknote.llm.exceptions import LLMError, MissingAPIKeyError

# Map hunknote provider names to LiteLLM model prefixes
PROVIDER_PREFIX_MAP = {
    "anthropic": "",
    "openai": "",
    "google": "gemini/",
    "mistral": "mistral/",
    "cohere": "command-r-plus",
    "groq": "groq/",
    "openrouter": "openrouter/",
}


def get_litellm_model_name(provider: str, model: str) -> str:
    """Convert hunknote provider+model to LiteLLM model string."""
    prefix = PROVIDER_PREFIX_MAP.get(provider, "")
    if "/" in model and prefix:
        return f"{prefix}{model}"
    return f"{prefix}{model}" if prefix else model


def create_llm_call_fn(provider: str, model: str, api_key: str,
                       temperature: float = 0.3) -> callable:
    """Create (system_prompt, user_prompt) -> RawLLMResult callable."""
    litellm_model = get_litellm_model_name(provider, model)

    def llm_call_fn(system_prompt: str, user_prompt: str) -> RawLLMResult:
        start_time = time.time()
        try:
            response = litellm.completion(
                model=litellm_model,
                messages=[
                    {"role": "system", "content": system_prompt},
                    {"role": "user", "content": user_prompt},
                ],
                api_key=api_key,
                temperature=temperature,
                max_tokens=8192,
            )
        except litellm.AuthenticationError as e:
            raise MissingAPIKeyError(f"API key error for {provider}: {e}")
        except Exception as e:
            raise LLMError(f"LLM call failed ({provider}/{model}): {e}")

        raw_response = response.choices[0].message.content
        input_tokens = response.usage.prompt_tokens if response.usage else 0
        output_tokens = response.usage.completion_tokens if response.usage else 0
        thinking_tokens = 0
        if hasattr(response.usage, 'reasoning_tokens'):
            thinking_tokens = response.usage.reasoning_tokens or 0

        return RawLLMResult(
            raw_response=raw_response,
            model=model,
            input_tokens=input_tokens,
            output_tokens=output_tokens,
            thinking_tokens=thinking_tokens,
        )

    return llm_call_fn
```

## Module 15: Output Validation Gates

**File**: `hunknote/compose/agent/gates.py`

```python
@dataclass
class GateViolation:
    violation_type: str   # "frozen_hunk_referenced", "nonexistent_hunk", etc.
    hunk_id: str
    detail: str

def validate_clustering_output(
    groups: list[dict],
    mutable_hunk_ids: set[str],
    frozen_hunk_ids: set[str],
    max_commits: int,
) -> tuple[bool, list[GateViolation]]:
    """Validate Phase 3 output. Checks: no frozen hunks, no nonexistent hunks,
    no duplicates, no unassigned, no empty groups, count ≤ max."""
    violations = []
    all_assigned = []

    for group in groups:
        hunk_ids = group.get("hunks", [])
        if len(hunk_ids) == 0:
            violations.append(GateViolation("empty_group", "", f"Group {group.get('id', '?')} has no hunks"))

        for hid in hunk_ids:
            if hid in frozen_hunk_ids:
                violations.append(GateViolation(
                    "frozen_hunk_referenced", hid,
                    f"Hunk {hid} is frozen in a validated commit and cannot be moved",
                ))
            elif hid not in mutable_hunk_ids:
                violations.append(GateViolation(
                    "nonexistent_hunk", hid,
                    f"Hunk {hid} does not exist in the inventory",
                ))
            if hid in all_assigned:
                violations.append(GateViolation(
                    "duplicate_assignment", hid,
                    f"Hunk {hid} assigned to multiple groups",
                ))
            all_assigned.append(hid)

    # Check for unassigned mutable hunks
    assigned_set = set(all_assigned)
    for hid in mutable_hunk_ids:
        if hid not in assigned_set:
            violations.append(GateViolation(
                "mutable_hunk_unassigned", hid,
                f"Hunk {hid} was not assigned to any group",
            ))

    # Check commit count
    if len(groups) > max_commits:
        violations.append(GateViolation(
            "too_many_groups", "",
            f"Plan has {len(groups)} groups, exceeds max of {max_commits}",
        ))

    return (len(violations) == 0, violations)


def validate_remediation_output(
    remediation: dict,
    mutable_hunk_ids: set[str],
    frozen_hunk_ids: set[str],
    inventory_hunk_ids: set[str],
) -> tuple[bool, list[GateViolation]]:
    """Validate Phase 5b output. Checks: valid hunk IDs, no frozen hunks
    in non-unlock actions, no self-dependencies."""
    violations = []
    action = remediation.get("action", "")

    # Check all hunk IDs in modifications
    mods = remediation.get("modifications", {})

    for group in mods.get("merge_hunks_into_same_commit", []):
        for hid in group:
            if hid not in inventory_hunk_ids:
                violations.append(GateViolation(
                    "nonexistent_hunk_in_remediation", hid,
                    f"Hunk {hid} does not exist",
                ))
            elif hid in frozen_hunk_ids and action != "unlock_and_recluster":
                violations.append(GateViolation(
                    "frozen_hunk_in_remediation", hid,
                    f"Hunk {hid} is frozen — use unlock_and_recluster to modify frozen commits",
                ))

    for edge in mods.get("add_dependency_edges", []):
        for key in ("from", "to"):
            hid = edge.get(key, "")
            if hid not in inventory_hunk_ids:
                violations.append(GateViolation(
                    "nonexistent_hunk_in_edge", hid,
                    f"Edge references nonexistent hunk {hid}",
                ))
            if hid == edge.get("from") == edge.get("to"):
                violations.append(GateViolation(
                    "self_dependency", hid,
                    f"Self-referential dependency edge on {hid}",
                ))

    return (len(violations) == 0, violations)


def build_correction_prompt(
    violations: list[GateViolation],
    mutable_hunk_ids: set[str],
) -> str:
    """Build specific reprompt with violations and valid hunk ID list.
    Hunk IDs sorted by numeric prefix for readability."""
    lines = ["Your previous output was invalid. The following violations were detected:", ""]
    for i, v in enumerate(violations, 1):
        lines.append(f"VIOLATION {i}: [{v.violation_type}] {v.detail}")
    lines.append("")
    lines.append("VALID HUNK IDs you may use:")
    lines.append(", ".join(sorted(mutable_hunk_ids)))
    lines.append("")
    lines.append("Please provide a corrected output.")
    return "\n".join(lines)
```

2. **Tracing**: Test nested event emission, JSON serialization/deserialization, save/load from file, summary generation, format_for_stderr with verbose and compact modes.
3. **LiteLLM wrapper**: Test `get_litellm_model_name` mapping, mock `litellm.completion`, test error handling (auth errors, timeouts).
4. **Tools**: Mock `subprocess.run` and test each tool function. Test truncation behavior. Test fallback from `rg` to `grep`.
5. **Validation Gates**: Test all violation types with fixture data. This is the most critical test — use parameterized tests. Test `build_correction_prompt` output.
6. **Batching**: Test with small files (should merge), large files (should split), new files, mixed sizes. Test token estimation.
7. **ReAct Engine**: Mock `llm_call_fn`, test parsing of Thought/Action/Output blocks, test iteration limits, test nudge message.
8. **Phase 1**: Mock LLM, test JSON parsing, test new-file handling, test fallback on parse failure.
9. **Phase 3**: Mock LLM, test gate enforcement (frozen hunk rejection, reprompting).
10. **Phase 4**: Test topological sort with known DAGs. Test cycle detection. Test C{n} ID assignment.
11. **Phase 5a**: Test worktree creation/cleanup at `~/.hunknote/tmp/`. Mock git commands. Test each validation layer independently.
12. **Orchestrator**: Integration test with all phases mocked. Test the retry loop, frozen/mutable state transitions, remediation application, trace save after each phase.

## Additional Notes for Implementation

1. **Do not modify any existing files in `hunknote/compose/` except `__init__.py` and the backward compat shim `hunknote/compose.py`** (to remove relationship exports). All new code goes in `hunknote/compose/agent/`.

2. **The `compose_command` in `cli/compose.py` needs minimal changes**: the `--agent` and `--trace` flags, the branch point, trace cleanup on commit, and removal of `relationships.py` imports.

3. **The agent's `ComposePlan` output must be identical in structure to the existing single-shot output.** This means the existing caching, rendering, display, and execution code works unchanged.

4. **Token budget tracking**: The `AgentTrace` accumulates total tokens across all LLM calls. Displayed in the compact stderr summary and stored in cache metadata.

5. **`--debug` vs `--trace`**: The `--debug` flag retains its existing behavior (showing compose debug info like cache status, diff statistics, file list, etc.). The `--trace` flag is new and shows the full agent trace (every LLM call, tool call, validation check, and state change). Both can be used together: `hunknote compose --agent --debug --trace`.

6. **Trace file lifecycle**: The trace JSON file at `.hunknote/agent_trace.json` is:
   - Created/overwritten when the agent pipeline starts.
   - Updated incrementally after each phase completes.
   - Preserved after the pipeline finishes (whether success or failure) for post-mortem debugging.
   - Cleared only when the compose plan is committed (alongside other compose cache files).

7. **Worktree path**: All temporary worktrees for Phase 5a validation use `~/.hunknote/tmp/` as the base directory, NOT `/tmp/`. This keeps temporary files in the user's hunknote directory and avoids issues with `/tmp` cleanup policies on some systems.

8. **New file handling**: Files with `is_new_file=True` get special treatment:
    - Phase 1: Summarized as a whole-file unit, not per-hunk diff analysis.
    - Phase 2: All hunks auto-linked with bidirectional edges (must stay in same commit). Agent skips intra-file dependency investigation, focuses only on cross-file dependencies.
    - Phase 3: The bidirectional edges from Phase 2 force all new-file hunks into the same commit group automatically.

9. **LiteLLM dependency**: Add `litellm` to the project's dependencies (`pyproject.toml`). It is lazy-imported only when `--agent` is used, so it does not affect the non-agent flow.

10. **Existing `compose/validation.py`** (`validate_plan`, `try_correct_hunk_ids`): Keep these. They are still used for the non-agent flow. The agent's `gates.py` is a separate validation layer for LLM outputs during the agent pipeline.

11. **Error handling**: Every phase function should catch LLM errors and re-raise as `AgentPipelineError` with a clear message. The orchestrator catches this and surfaces it to the user.

12. **`compose/relationships.py` removal**: This file is imported in 3 places. Remove the imports and all code that calls `detect_file_relationships()` or `format_relationships_for_llm()`. The agent's Phase 2 replaces this functionality entirely.
